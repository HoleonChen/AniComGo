0. 项目全局背景（三人通用）
使用方法： 在开启任何新对话前，先发送这段背景，让 AI 了解你们的极限约束。

“我们正在进行一个为期 2 周的 Web 综合实践课。项目是一个二次元番剧点评网站。技术栈：前端 React (Vite) + Ant Design，后端 Spring Boot + MyBatis-Plus，数据库 MySQL 8.0。团队成员多为零基础，我是[前端/后端/数据库]工程师，请在后续回答中务必提供最简捷、生产力最高的代码方案，优先使用自动生成工具和成熟组件库。”


1. 数据库工程师专用 Prompt
核心任务：建表、维护数据字典、提供测试数据

“你现在是本项目的数据架构师。请根据以下需求，为我生成 MySQL 8.0 的建表语句（DDL）：

表结构： 包含用户表（需区分权限和状态）、番剧表、标签表、角色表、声优表、工作室表、收藏/评分表、评论表（支持楼中楼嵌套）。

约束： 处理好番剧与标签、角色、工作室的多对多关系（建立中间表）。

细节： >    - 所有表必须包含 id (主键), created_at, updated_at。

评论表内容必须支持 Emoji (utf8mb4)。

请为每张表提供 5 条具有‘二次元特色’的初始测试数据（DML），方便前后端联调。

产出： 请直接给出完整的 SQL 脚本。”


2. 后端工程师专用 Prompt
核心任务：搭建框架、写 CRUD 接口、与前端联调

“你现在是高级 Java 开发工程师。我将使用 Spring Boot + MyBatis-Plus 开发番剧站后端。

架构要求： 请帮我初始化项目目录结构，并配置 pom.xml（包含 Web, MySQL Driver, Lombok, MyBatis-Plus, Spring Security 基础依赖）。

代码生成： 我会给你数据库建表语句，请告诉我如何利用 MyBatis-Plus 的 AutoGenerator 快速生成 Entity、Mapper、Service 和 Controller。

接口规范： 所有的控制器接口必须返回统一的 JSON 格式（包含 code, message, data）。

核心逻辑： 请先帮我实现‘番剧分页查询’和‘用户登录（加密存储）’的示例代码，代码要尽量简洁，避免过度封装。”


3. 前端工程师专用 Prompt
核心任务：组件化开发、UI 布局、接口对接

“你现在是资深前端架构师。我有 iOS/Android 经验，正在用 React + Vite + Ant Design 开发番剧站。

基础布局： 请用 Ant Design 的 ProLayout 或常用组件帮我写一个响应式的导航框架。

核心页面： 请提供一个‘番剧列表页’的代码示例，要求使用 Card 组件展示番剧海报和标题，并包含一个搜索框。

网络请求： 请帮我封装一个 axios 实例，包含基础路径配置和简单的请求/响应拦截器。

提速策略： 既然没有设计师，请多推荐 Ant Design 中现成的组件来完成‘评分（Rate）’、‘进度管理（Slider/Steps）’和‘评论区’的功能实现。”
